1 Introduction

1.1 Purpose

This is the Product Requirements Specification for COPYStation2 (CS2_PRS). It provides an overview of the system and the
device and specifies the top level product requirements.

1.2 Scope

The requirements herein focus on the functionality visible to the user/customer without specifying implementation details, i.e.
technical requirements. The implementation details can be found in [CS2_TRS].

2 System Overview

The COPYstation2 is a simple-to-use adapter from IPETRONIK cartridges to Customer IT infrastructure using high speed Ethernet

connections.

Figure 1 COPYstation2 enclosure concept

It is meant to be mounted on a server rack, physically attached to a customer's IT infrastructure, and provides slots for hot-plugging
cartridges, as well as a display and LEDs for status presentation and buttons for headless control, i.e. control without any external
display.

vvHLP-1446 - Testing - HIL tests

The COPYstation2 HW must provide access to inputs and outputs with minimal modifications to allow integration into a HIL test

setup which can, at a minimum, simulate the insertion and removal of a cartridge while the actual physical cartridge remains

inserted.

Rationale: The project is expected to be further improved after a first release in an agile process. Such a process requires

automated regression tests to ensure that new features don't break existing ones. One of the main functions of the COPYstation2

is to autodetect and copy datasets available on a cartridge upon its insertion. Without a way to simulate the insertion and removal

of cartridges, the functionality cannot be automatically tested at system level. Manual tests are possible, but not cost-effective for

an agile development with continuous releases.
